# 構造体

```
// 構造体を定義
#[allow(dead_code)]
struct User {
    username: String,
    email: String,
    sign_in_count: u64,
    active: bool,
}

fn main() {
    // インスタンス化
    // 各フィールドに対して具体的な値を指定
    // フィールドは、 構造体で宣言した通りの順番に指定する必要はありません
    let mut user1 = User {
        email: String::from("someone@example.com"),
        active: true,
        sign_in_count: 1,
        username: String::from("someusername123"),
    };

    // 構造体から特定の値を得るには、ドット記法
    println!("{}", user1.email);

    // インスタンスが可変であれば、ドット記法を使い特定のフィールドに代入することで値を変更できます
    user1.username = String::from("someusername999");
    println!("{}", user1.username);
}
```

    // フィールドと変数が同名の時は省略できる
    let user1 = User {
        username,
        email,
        active: true,

    // ここではもう使えない
    // println!("{}", username);
    // println!("{}", email);

```

        username: String::from("anotherusername567"),
        ..user1
    };

    println!("{:?}", user2);
}

### 構造体を使ったプログラム例

何故構造体を使うか？  
構造体で複数データを意味付けしてプログラムの意図を明瞭にできる

https://doc.rust-jp.rs/book/second-edition/ch05-02-example-structs.html


## メソッドを定義する

rect1.area()として関数を呼び出せる
```
#[derive(Debug)]
struct Rectangle {
    width: u32,
    height: u32,
}

// implで囲う
impl Rectangle {
    // impl内でselfや&selfを引数にする関数がメソッド
    fn area(&self) -> u32 {
        self.width * self.height
    }
}

fn main() {
    let rect1 = Rectangle {
        width: 30,
        height: 50,
    };

    println!(
        "The area of the rectangle is {} square pixels.",
        rect1.area()
    );

    // fn area(&self) -> u32{}
    // areaメソッドの引数がselfならばrect1はムーブして下はエラーになる
    println!("{:?}", rect1);
}
```
&mut self のメソッド例
```
#[derive(Debug)]
struct Rectangle {
    width: u32,
    height: u32,
}

// implで囲う
impl Rectangle {
    // impl内でselfや&selfを引数にする関数がメソッド
    fn area(&self) -> u32 {
        self.width * self.height
    }

    // 中身を更新するメソッドの場合は&mut self
    fn update(&mut self, n: u32) {
        self.width = n;
        self.height = n;
    }
}

fn main() {
    // 更新できるようにmutにする
    let mut rect1 = Rectangle {
        width: 30,
        height: 50,
    };

    rect1.update(40);

    println!(
        "The area of the rectangle is {} square pixels.",
        rect1.area()
    );
}
```

#[derive(Debug)]
struct Rectangle {

### 関連関数

implブロック内にselfを引数に取らない関数を定義できる。  
これは構造体に関連付けられているので、関連関数と呼ばれます。  
関連関数は、構造体の新規インスタンスを返す関数としてよく使用されます。
```
struct Rectangle {
    width: u32,
    height: u32,
}

impl Rectangle {
    // 関連関数
    fn square(size: u32) -> Rectangle {
        Rectangle {
            width: size,
            height: size,
        }
    }

    // メソッド
    fn area(&self) -> u32 {
        self.width * self.height
    }
}

fn main() {
    // 関連関数の呼び出し
    let rect1 = Rectangle::square(10);
    // メソッドの呼び出し
    println!("{}", rect1.area());
}
```
こう書けますが、implブロックに関連のある関数をまとめた方が見やすいということ
```
struct Rectangle {
    width: u32,
    height: u32,
}

impl Rectangle {
    // メソッド
    fn area(&self) -> u32 {
        self.width * self.height
    }
}

// 普通の関連として定義できる
fn square(size: u32) -> Rectangle {
    Rectangle {
        width: size,
        height: size,
    }
}

fn main() {
    // 普通の関数の呼び出し
    let rect1 = square(10);
    // メソッドの呼び出し
    println!("{}", rect1.area());
}
```

    println!("{}", integer_and_float.y);
}
```

## ジェネリックな構造体のメソッド定義

implの直後にTを宣言
```
#[allow(dead_code)]
struct Point<T> {
    x: T,
    y: T,
}

impl<T> Point<T> {
    fn x(&self) -> &T {
        &self.x
    }
}

fn main() {
    let p = Point { x: 5, y: 10 };
    let p2 = Point { x: 5.01, y: 3.14 };
    println!("{}", p.x());
    println!("{}", p2.x());
}
```
複雑なジェネリックの場合
```
struct Point<T, U> {
    x: T,
    y: U,
}

impl<T, U> Point<T, U> {
    fn mixup<V, W>(self, other: Point<V, W>) -> Point<T, W> {
        Point {
            x: self.x,
            y: other.y,
        }
    }
}

fn main() {
    let p1 = Point { x: 5, y: 10.4 };
    let p2 = Point { x: "Hello", y: 'c'};

    let p3 = p1.mixup(p2);

    println!("p3.x = {}, p3.y = {}", p3.x, p3.y);
}
```

# トレイト

トレイトは色々な「型」に同じようなメソッドを追加するものです。

```
// トレイト
trait HasArea {
    // メソッド本体を定義しない(定義してもいい)
    fn area(&self) -> f64;
}

// 長方形
struct Rectangle {
    vertical: f64,   //縦
    horizontal: f64, //横
}
// トレイトを実装
impl HasArea for Rectangle {
    fn area(&self) -> f64 {
        self.vertical * self.horizontal
    }
}

// 三角形
struct Triangle {
    bottom: f64, //底辺
    height: f64, //高さ
}
// トレイトを実装
impl HasArea for Triangle {
    fn area(&self) -> f64 {
        self.bottom * self.height / 2.0
    }
}

// 円
struct Circle {
    radius: f64, //半径
}
// トレイトを実装
impl HasArea for Circle {
    fn area(&self) -> f64 {
        self.radius * self.radius * 3.14
    }
}

fn main() {
    let a1: Rectangle = Rectangle {
        vertical: 10.,
        horizontal: 20.,
    };
    let a2: Triangle = Triangle {
        bottom: 30.,
        height: 20.,
    };
    let a3: Circle = Circle { radius: 30. };

    println!("{}", a1.area());
    println!("{}", a2.area());
    println!("{}", a3.area());
}
```

違う型に共通のトレイトを実装することにより、それらが同じ型になったような(実際は同じ型でない)ことができます。
```
// トレイト
trait HasArea {
    // メソッド本体を定義しない(定義してもいい)
    fn area(&self) -> f64;
}

// 長方形
struct Rectangle {
    vertical: f64,   //縦
    horizontal: f64, //横
}
// トレイトを実装
impl HasArea for Rectangle {
    fn area(&self) -> f64 {
        self.vertical * self.horizontal
    }
}

// 三角形
struct Triangle {
    bottom: f64, //底辺
    height: f64, //高さ
}
// トレイトを実装
impl HasArea for Triangle {
    fn area(&self) -> f64 {
        self.bottom * self.height / 2.0
    }
}

// 円
struct Circle {
    radius: f64, //半径
}
// トレイトを実装
impl HasArea for Circle {
    fn area(&self) -> f64 {
        self.radius * self.radius * 3.14
    }
}

// ジェネリックT型はあらゆる型
// かつarea メソッドが実装されていること
// トレイト境界の追加でHasAreaトレイトを実装している型のみ受け入れる
fn print_area<T: HasArea>(shape: &T) {
    println!("{}", shape.area());
}

fn main() {
    let a1: Rectangle = Rectangle {
        vertical: 10.,
        horizontal: 20.,
    };
    let a2: Triangle = Triangle {
        bottom: 30.,
        height: 20.,
    };
    let a3: Circle = Circle { radius: 30. };

    // 型が違っても同じ関数を実行できる
    // これはトレイト境界をつかっているので「静的ディスパッチ」と呼ぶ
    print_area(&a1);
    print_area(&a2);
    print_area(&a3);
}
```
動的ディスパッチ  
記述は簡単だが静的ディスパッチよりも低速  
これは他の言語(Java、C#など)のインターフェースをつかったポリモーフィズムに近い

トレイトオブジェクトを入れたい  
構造体を作り、その中に`Vec<Box<dyn Animal>>`型のフィールドを持つ方法
```
trait Animal {
    fn say(&self) -> String;
}

struct Together {
    obj: Vec<Box<dyn Animal>>,
}

struct Dog {
    name: String,
}

impl Dog {
    fn my_name(&self) -> String {
        self.name.to_string()
    }
}

impl Animal for Dog {
    fn say(&self) -> String {
        format!("{} ワンワン", self.my_name())
    }
}

struct Cat {
    name: String,
}

impl Cat {
    fn my_name(&self) -> String {
        self.name.to_string()
    }
}

impl Animal for Cat {
    fn say(&self) -> String {
        format!("{} にゃん", self.my_name())
    }
}

impl Together {
    fn run(&self) {
        for i in self.obj.iter() {
            println!("{}", i.say());
        }
    }
}

fn main() {
    let mut s = Together {
        obj: vec![
            Box::new(Dog {
                name: "ぽち".to_string(),
            }),
            Box::new(Cat {
                name: "たま".to_string(),
            }),
        ],
    };

    s.obj.push(Box::new(Cat {
        name: "くろ".to_string(),
    }));

    s.run();
}
```
